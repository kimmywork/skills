我新建了一个plugin bundle @loop/ 主要是为了做一个提供设计自动化 workflow 的创建过程和工具。其中一部分思路来自于 @docs/references/loop-engineering.md 。这个思路的另一个成果是我们集成起来的 @delivery/ 下面的skill集。

但是这个 loop creation kit 我希望换一个思路。首先，我希望的是有一个核心的技能 loopify 来帮我创建这种固定触发的、重复执行的、有明确的工作流程的工作流。并且可以通过 loopy 来定期执行这些工作流。loopy的核心就是一个脚本，检查下面的workflow schedule（daily / weekly / or hourly，or a cron expression），然后按照 workflow 里的步骤描述来执行。

这样会产生两个skill： loopify 创建 workflow， loopy 执行 workflow。同时我希望创建过程是一个agent-assisted的过程，用户通过描述自己的工作流程，然后由Agent不断追问以确定是否可以细化到workflow级别的spec——有清晰明确的步骤、固定的触发条件、稳定的输入输出、每一步都可执行可验证、能够追溯并且可以重试/恢复等等。这样一个spec会作为一个 markdown 文档放置在 workflows/ 下面，markdown的front-matter包含对应的schedule（定期间隔/时间点触发）或者触发条件（比如收到 email / 新的Pull Request / 检查网络中断等等）。当 loopy skill执行时，agent 可以依据 workflow 的信息来进行workflow里描述相应的工作，完成一个个循环。

Workflow没有硬性的结构要求，核心只有一个触发规则，具体的内容只要agent能够确定足够清晰没有疑问就可以。

你可以追问我各种问题了（grilling）。
