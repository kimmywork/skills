# Increment Record

Use after each Build increment. The record is evidence for Verify and Record.

```markdown
## Increment: <name>

- Parent track / stage:
- Scope item:
- Acceptance criteria covered:
- Expected outcome:
- Changes made:
- Changed artifacts:
- Contracts touched:
- Commands/checks run:
- Evidence:
- Failures/warnings:
- Drift or limitations:
- Follow-up required:
- Next increment:
```

## Software evidence

Include exact commands and results:

```markdown
- `npm test`: pass/fail, summary
- `npm run build`: pass/fail, summary
- Manual QA: scenario, result
```

## Writing/research evidence

Include:
- sections drafted or revised;
- claims added and their sources;
- style profile used;
- fact checks performed;
- unresolved claims.

## Rule

If evidence is missing, mark it missing. Do not convert missing evidence into confidence language.
