# Plan Execution

Build only the approved thing and preserve verification evidence.

## Load

- `references/increment-record.md`
- `references/execution-control.md`
- `references/software-mode.md` for code/config/infrastructure work
- `../drafting-loop/references/change-note-template.md`
- `../drafting-loop/references/iron-rules.md`

## Preconditions

- Scope exists.
- Blueprint exists unless the kind allows fast-pass.
- Acceptance criteria and verification method are known.

If a precondition is missing, stop and shape/design rather than inventing it.

## Workflow

1. Perform the before-editing scan from `references/execution-control.md`.
2. Restate active increment, contracts, acceptance criteria, and evidence.
3. Execute the smallest verifiable increment.
4. Record changed artifacts and evidence.
5. Run planned checks or the closest available checks.
6. Apply stop conditions, blocker handling, scope-map updates, and change control from `references/execution-control.md`.
7. Hand off to Verify with deliverables, evidence, drift, and limitations.

## Kind-specific execution

- Crafting: use software-mode when relevant; verify dependents and contracts.
- Composing: draft sections against brief, cite sources, preserve audience and style.
- Evaluating: use `../audit-trail/SKILL.md` instead when evaluation is the deliverable.
- Investigating: use `../deep-research/SKILL.md` when discovery is the deliverable.
- Creating: produce draft options, then converge against constraints.

## Output

End with an increment record and a Build -> Verify handoff.

## Language

Human-facing output follows the user's language. Durable skill artifacts stay English.
