# Scope Shaping

Shape the work so it can be executed, reviewed, or handed off without inventing the target.

## Load

- `references/discovery-process.md`
- `references/scope-templates.md`
- `references/requirement-syntax.md`

- `../drafting-loop/references/track-document-structure.md`
- `../drafting-loop/references/multi-stage-tracks.md` when scope must split
- `../drafting-loop/references/handoff-schemas.md`

## Workflow

1. Infer kind and mode from the request and materials; if the target is still not stable enough to scope, ask only the questions that unblock a usable scope.
2. Run material scan, intent shaping, challenge pass, and output-scale selection from `references/discovery-process.md`.
3. Accept partial inputs such as notes, chats, or existing docs, then reorganize them into an explicit scope.
4. Check active tracks for overlapping scope before creating a new one.
5. Write in-scope and out-of-scope lists.
6. Convert desires into observable acceptance criteria using User Story, EARS, or Given/When/Then when helpful.
7. Split broad work into independently verifiable stages.
8. Add a verification plan for each criterion.
9. Flag open questions; ask only if they block safe progress.
10. Produce a scope artifact ready for design, execution, review, or research.

## Kind-specific shaping

- Crafting: users/jobs, behavior, contracts, constraints, acceptance tests, non-goals.
- Composing: audience, purpose, message, sources, structure, style, completeness checks.
- Evaluating: artifact, criteria, severity definitions, evidence requirements, verdict taxonomy.
- Investigating: question, boundaries, source hierarchy, confidence labels, expected synthesis.
- Creating: premise, constraints, references, style direction, success criteria.

## Quality bar

A scope is ready when another agent could execute, review, or design from it without inventing the target.

## Output

Use this structure: Goal, Kind/mode, Materials, In scope, Out of scope, Acceptance criteria, Verification plan, Risks, Open questions.

## Language

Human-facing output follows the user's language. Durable skill artifacts stay English.
